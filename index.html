Part I

Theory

What is Space Syntax?

"Space Syntax is a theory and a set of methods about space, built on two ideas which reflect both the objectivity of space and our intuitive engagement with it."

Hillier and Vaughan (2007)

Objectivity of space
Space has measurable, structural properties — configuration — that exist independently of how any individual perceives them.
Intuitive engagement
People navigate and experience space through movement, interaction, and vision — not through abstract plans.

Foundation 1: Space is not the background

"To think of space not as the background to human activity, as we think of it as the background to objects, but as an intrinsic aspect of everything human beings do."

— Hillier (2005)  ·  People: move in lines · interact in convex spaces · see changing visual fields

Foundation 2: Configuration of Space

"Human space is not just about the properties of individual spaces, but about the inter-relations between the many spaces that make up a spatial layout of a building or a city as a whole."

— Hillier (2005)

This is what we call configuration: the simultaneously existing relations among all parts that make up the whole.

Depth: exists wherever it is necessary to pass through intervening spaces to get from one space to another.

Convex Spaces

A convex space is a space in which all points can see all other points.

A convex map divides a building or urban area into the:

  • Fewest (least number) and
  • Fattest (largest area) possible convex spaces

Convex spaces form the nodes of a graph; adjacency between spaces forms the connections. This is the foundation for computing integration.

The Axial Model

Axial lines are the longest lines of sight in a spatial configuration. The axial map draws the fewest intersecting lines through all spaces of the urban grid so that the grid is fully covered and all rings of circulation are completed.

The Axial Graph (Dual Graph)

The dual graph (or axial graph) is converted from the axial map:

  • Each axial line → node
  • Each intersection → connection between nodes

Where two roads cross at a bridge or underpass without connecting, an unlink removes that connection from the graph.

Integration & Choice

Integration — to-movement
Based on closeness centrality (Sabidussi, 1966). Measures how close a line is to all others in the network — streets with high integration are easy to reach from everywhere.
Shortest path = fewest topological steps.
Choice — through-movement
Based on betweenness centrality (Freeman, 1977). Counts the number of shortest paths between all pairs of lines that pass through a given line — streets with high choice are natural through-routes.
Shortest path = fewest topological steps.

Radius = n (global) or a fixed number of topological steps (local). Axial analysis uses topological distance only.

Extending to Consider Angles

Axial line analysis does not consider the angle at which streets connect. Seen from line 1, connecting to lines 2, 3 or 4 is treated equally regardless of turn angle. To measure angular difference we split lines at intersections — this is what the segment model does.

The Segment Model

Starting from an axial line map, the segment map is built by dividing each axial line at every point of intersection, producing shorter units called segments.

Short end-pieces called stubs — axial line remainders shorter than 40% of the original line length — are removed from the map.

In segment maps, only lines that share a node are connected in the dual graph (unlike axial maps where any overlap creates a connection).

Normalised Measures: NAIN & NACH

Because raw values grow exponentially with network size, Hillier et al. (2012) introduced normalised measures to allow comparison across different models:

NAIN
Normalised Angular Integration
To-movement: how accessible a segment is as a destination. \[ \text{NAIN} = \frac{NC^{1.2}}{TD + 2} \] NC = node count · TD = total angular depth
NACH
Normalised Angular Choice
Through-movement: how likely a segment is to be used as a route. \[ \text{NACH} = \frac{\log(AC + 1)}{\log(TD + 3)} \] AC = angular choice · TD = total angular depth

Dead-end segments return NACH = −1 and should be recoded to 0.

Radius & Scale

Angular Segment Analysis offers three distance types for radius:

Metric
Cut-off in metres. Most common in practice.
Topological
Number of segment-to-segment connections traversed.
Angular
Cumulative turn angle as the cut-off.

Most studies use metric radius, computed at multiple scales simultaneously:

R800 · R1600 · R5000 · Rn

R1600 ≈ walkable distance

Smaller radii reveal neighbourhood structure; larger radii reveal the city-wide network hierarchy.

Axial vs. Angular Segment Analysis

Axial Analysis (AA) Angular Segment Analysis (ASA)
Approach Topological Geometric
Unit Longest lines of sight Segments (axial lines split at junctions)
Shortest path Fewest topological steps Smallest cumulative angular turn
Distance types Topological steps only Metric, topological, or angular
Best for How we conceive space How people move through space
Key measures Integration [HH], Choice NAIN, NACH

STEP 1 Download OSM Street Network

We use OpenStreetMap (OSM) as our street network data source — freely available, global coverage, and regularly updated by the community.

STEP 1 What is OpenStreetMap?

OpenStreetMap is a collaborative, open geographic database. For space syntax work, we use the highway tag to extract the street network.

Relevant OSM highway values:

  • motorway, trunk, primary
  • secondary, tertiary
  • residential, living_street
  • footway, pedestrian, path

Tip: For segment analysis, include all pedestrian-accessible streets.

STEP 1 Downloading via QuickOSM in QGIS

  1. Open QGIS → Plugins → Manage and Install Plugins → search QuickOSM → Install
  2. Go to Vector → QuickOSM → QuickOSM
  3. In the Quick Query tab:
    • Key: highway  |  Value: (leave empty for all)
    • Set extent: Map Canvas Extent or type a place name
  4. Click Run query — OSM will return lines, points, and multipolygons
  5. Keep only the lines layer (this is the street network)
Note: OSM data for large cities can be slow to download. Alternatively, download a pre-clipped .osm.pbf from Geofabrik and load it with the OSM Tools plugin.

STEP 1 Inspecting the Raw Data

After downloading, open the Attribute Table and check:

  • The highway field — filter out motorways if doing pedestrian analysis
  • Presence of oneway tags (relevant for vehicle analysis)
  • Geometry: lines should be single-part, not multi-part

Use Vector → Geometry Tools → Multipart to Singleparts if needed.

STEP 2 Simplify the Road Centre Line

Raw OSM data contains topological errors, duplicate edges, and unnecessary complexity that must be cleaned before space syntax analysis.

STEP 2 Why Simplification Matters

Space syntax algorithms require a clean, planar graph: every line must only intersect another at shared endpoints (nodes). Common issues in raw OSM data:

Overshoots / Undershoots
Lines that do not quite meet at a junction, leaving gaps or overlaps.
Duplicate roads
Dual carriageways drawn as two parallel lines instead of one centre line.
Redundant vertices
Curves represented by many densely spaced points that inflate the graph.

STEP 2 Cleaning Steps in QGIS

  1. Filter road types
    Use Select by Expression to remove motorways, service roads, or other irrelevant categories for your analysis scope.
  2. Dissolve & Split at intersections
    Run Vector → Geoprocessing Tools → Dissolve (by highway type if needed), then use Processing → Split with Lines or the GRASS v.clean tool to split all lines at every crossing.
  3. Remove small dangles
    Use GRASS v.clean → tool rmsa (remove small areas) and break to fix topology. Set a snap tolerance appropriate to your data (typically 0.5–2 m).
  4. Verify planarity
    Use Vector → Geometry Tools → Check Validity and confirm there are no geometry errors before proceeding.

STEP 2 Using the SST Unlinks Tool

The Space Syntax Toolkit includes an Unlinks tool to handle bridges and underpasses — where two roads cross in 3D but do not connect.

After cleaning:

  1. Generate unlinks using SST → Network → Generate Unlinks
  2. Manually inspect and remove false unlinks
  3. Save the cleaned network and unlinks as separate layers

STEP 3 Calculate Space Syntax Models

With a clean network, we use the Space Syntax Toolkit (SST) QGIS plugin to compute Integration and Choice at multiple radii.

STEP 3 Installing the SST Plugin

  1. Open QGIS → Plugins → Manage and Install Plugins
  2. Search for "Space Syntax Toolkit" → Install
  3. After installation, a new SST toolbar and menu will appear
  4. Open the SST panel: SST → Open Space Syntax Toolkit
Requirements: QGIS 3.x · Python 3 · The plugin uses NetworkX for graph computation — no separate installation needed if QGIS was installed via OSGeo4W or the standard installer.

STEP 3 Running Axial Analysis

  1. In the SST panel, go to the Analysis tab
  2. Select Analysis type: Axial / All-line
  3. Set Map layer: your cleaned axial or road centre-line layer
  4. Set Unlinks layer: the unlinks layer generated in Step 2 (if applicable)
  5. Choose Radii: enter comma-separated values, e.g. n, 3, 5 (topological steps)
  6. Tick Integration and Choice
  7. Click Run — results are added as new fields to the network layer

STEP 3 Running Segment Analysis

  1. In the SST panel, select Analysis type: Segment
  2. Set Map layer: your cleaned segment network
  3. Choose Radii: metric radii in metres, e.g. 800, 1600, 5000, n
  4. Select Distance: Angular (default for NAIN/NACH)
  5. Tick NAIN and NACH
  6. Click Run
Output fields naming convention:
NAIN_R800 = Normalised Angular Integration at 800 m radius
NACH_Rn = Normalised Angular Choice at global radius

STEP 3 Visualising Results

Apply a graduated colour ramp in QGIS to visualise the results:

  1. Right-click layer → Properties → Symbology
  2. Select Graduated → choose field NAIN_Rn
  3. Use a spectral or red-blue colour ramp
  4. Set mode: Natural Breaks (Jenks) or Quantile
  5. Classify → OK

Convention: Red = high integration/choice · Blue = low

STEP 3 Interpreting the Output

High NAIN (Integration)
Streets that are topographically central — easy to reach from everywhere. Typically associated with retail frontage, pedestrian activity, higher land values.
High NACH (Choice)
Streets on the most-used through-routes. Associated with vehicle flows, bus routes, commercial corridors.

Compare results at different radii to understand how the network functions at pedestrian scale (R800) versus city scale (Rn).

A typical finding: global integration highlights the primary urban spine; local integration reveals neighbourhood centres and high streets.

Summary

  • Space Syntax analyses spatial configuration as a driver of movement and urban life
  • The axial model captures global topological structure using long sight lines
  • The segment model provides finer resolution using angular cost — expressed as NAIN and NACH
  • With OSM + QGIS + SST, the full workflow enables spatial analysis based on open data
